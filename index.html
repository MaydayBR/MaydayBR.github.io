My Work

Knomedic

Knowmedic is a mobile app that helps people discover and understand new medical research. It pulls recent PubMed studies, applies semantic search and ranking, and lets users save, organize, and interact with articles. The stack includes React Native (Expo), Firebase (Auth, Firestore, Cloud Functions, Storage), and Google Cloud (Pub/Sub, Cloud Scheduler, BigQuery) for ingestion, search, and scalable data pipelines.

Features

  • Bookmark system with custom folders to organize saved studies.
  • Unique-username reservation at sign-up with public profile pages and searchable profiles (users choose public vs. private during signup).
  • Hot New Studies tab powered by the PubMed API and Google Pub/Sub, showcasing recently published research.
  • Engagement-driven Trending tab where scores are dynamically boosted by likes and bookmarks.
  • Secure account deletion via a callable Cloud Function that recursively removes all user data.
  • Real-time updates using Firestore listeners for live feed changes (likes, comments, bookmarks).
  • Profile editing with bios
  • Firestore security rules enforcing per-user access and privacy controls.
  • Integration with Firebase Authentication for email/password login.
  • Serverless backend built on Google Cloud Functions, ensuring scalable and cost-efficient infrastructure.
  • Scheduled tasks (Pub/Sub + Cloud Scheduler) to refresh feeds, prune old data, and keep “Hot New Studies” current.

Stack

  • Frontend: React Native (Expo/EAS) with React Navigation; fetch-based API calls and real-time Firestore listeners for dynamic updates.
  • Firebase: Authentication (email/password), Firestore (profiles, bookmarks, articles, engagement data), Storage for media uploads, and Cloud Functions v2 (callable + trigger-based).
  • Google Cloud Platform: Pub/Sub (fan-out ingestion), Cloud Scheduler (cron jobs), Cloud Run/Functions for background workers, BigQuery for article text, metadata, and embeddings.
  • Data: PubMed eUtils (baseline + daily PMIDs), custom ingestion pipelines, OpenAI embeddings stored in BigQuery with IVF/HNSW vector indexes; hybrid keyword + semantic (vector) retrieval.
  • Tooling & Deployment: Git/GitHub for version control, GitHub Pages for Privacy Policy & Terms, EAS builds for app distribution, Xcode & Android Studio for local testing/debugging.
  • Other: Firestore security rules for per-user data isolation, recursive user deletion workflows, and Pub/Sub-driven trending/“Hot New” study pipelines.

Michigan Tech Week Hackathon Project

Here is a video of my hackathon project during Michigan Tech Week. I was the team lead for our project and I was the one who created the back end that you are seeing here. My team and I were given 10 days to try and come up with an idea to 'fix' a problem that Michigan is currently facing. The Michigan-based problem that we decided to tackle was the decreasing rates of physical activity in our state. Our idea was to create an easy-to-use and fun social way to create and plan local phycial activities/events. After making a social profile you are able to create or join local events such as a morning jog which your neighbor might have scheduled for 10 am tomorrow. Our final goal is to make it so that you are able to see the other profiles that have joined said event in your area and have a social web where you can see what all of your friends social plans are.
